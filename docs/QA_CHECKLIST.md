# 기능 점검 체크리스트 (QA)

- **대상 버전**: `v0.5.0`
- **범위**: 대화형 투두리스트 생성 데모 — 좌우 분할 UI(대화/체크리스트), 슬롯필링, 계획 생성/수정, mock 폴백, 여러 계획 보관(서버 공유 보관함), 오늘 할 일(오늘 보기)
- **사용법**: 배포 URL(`http://<PUBLIC_IP>` 또는 배포 주소)에 접속해 아래 항목을 순서대로 확인.
- **버전 갱신 시**: 새 버전에서 바뀐 항목을 추가하고 "대상 버전"을 올린다.

---

## A. 화면 / 레이아웃
- [ ] 좌우 분할이 보인다 — 왼쪽 **대화**, 오른쪽 **체크리스트**
- [ ] 헤더에 `DelayNoMore` + 우측 **AI 상태 표시**(점 + 텍스트)
- [ ] 초기 오른쪽 패널: "왼쪽 대화에서 … 입력하면 이곳에 계획표가 생성됩니다" 안내문
- [ ] 첫 진입 시 봇이 **첫 질문(목표)** 을 자동으로 발송

## B. AI 연결 상태 (헤더)
- [ ] 키 없이 배포 → **"API Key 미설정"**(주황) 표시 (생성은 mock 폴백)
- [ ] 키를 넣고 배포 → **"AI 연결됨"**(초록)

## C. 슬롯필링 대화 (순서대로 4개 질문)
- [ ] **목표** 입력 → 다음으로 **기간** 질문
- [ ] **기간** 질문 시 **3일/5일/7일 빠른 선택 버튼**이 **질문 말풍선 바로 아래(대화 영역 안)** 에 뜨고,
      클릭하면 바로 전송된다(직접 숫자 입력도 가능)
- [ ] **기간** 입력(버튼 또는 직접) → **하루 투자 시간** 질문 + **1/2/4/6시간 버튼**
- [ ] **하루 시간** 입력 → **현재 수준** 질문 + **완전 초보/기본 개념은 아는 수준/실전 경험 있음 버튼**
- [ ] 유저 메시지는 오른쪽 정렬(파란 말풍선), 봇은 왼쪽(회색)

## D. 입력 검증 (일부러 틀리게 입력)
- [ ] 목표에 `a` 한 글자 → "더 구체적으로" 경고
- [ ] 기간에 `0` 또는 `50` → "1~30일" 경고
- [ ] 기간/시간에 `abc`(숫자 아님) → 경고
- [ ] 하루 시간에 `30` → "1~24시간" 경고
- [ ] 수준에 한 글자 → 경고

## E. 체크리스트 생성 (핵심)
정상 입력 예: 목표 `React 기초 공부` / 기간 `3` / 시간 `2` / 수준 `완전 초보`
- [ ] 마지막 입력 후 **"분석 중… (n초)"** thinking 표시가 뜬다
- [ ] 오른쪽에 **제목 + 요약(기간·시간·수준)** 헤더 생성
- [ ] **Day 1 / Day 2 / Day 3** 카드가 날짜와 함께 표시
- [ ] 각 Day의 할 일 개수가 **하루 투자 시간에 비례**(1h→2개, 2h→3개, 3~4h→4개, 5h+→5~6개 안팎)
- [ ] 날짜가 **오늘부터** 순서대로 맞다

## F. 수정 루프 (초안 생성 후)
- [ ] 입력창 placeholder가 "수정 요청이나 질문을…"으로 바뀐다
- [ ] `주말은 빼줘` 입력 → 재생성 → 주말 처리 반영
- [ ] `일정을 늘려줘` 입력 → 할 일 개수/강도 증가
- [ ] 재생성 중에도 thinking 표시가 뜨고, 끝나면 오른쪽 갱신

## F-2. 자유 대화 (v0.1.0 포함, AI 키 연결 상태에서 확인 권장)
- [ ] 수정 요청 시 답변이 고정 문구가 아니라 **무엇을 어떻게 바꿨는지** 구체적으로 설명한다
- [ ] `공부 항목 좀 더 구체적으로` → 할 일 문구가 실제로 구체화되고, 답변에 변경 내용이 언급된다
- [ ] 계획에 대한 **질문**(예: "Day 2는 왜 이렇게 짰어?") → 계획을 다시 만들지 않고 자연어로 답한다
- [ ] `?` 같은 **불명확한 입력** → "반영했습니다" 대신 예시와 함께 되묻는다
- [ ] 직전 요청이 반영 안 됐을 때 `반영 안됐는데?` → 대화 이력을 참고해 그 요청을 다시 반영한다
- [ ] (키 없이 배포) 인식 못 하는 요청 → "(오프라인 모드)" 안내와 함께 가능한 요청 예시를 보여준다

## F-3. 체크리스트 UX (v0.2.0, 접근성 v0.4.1)
- [ ] 체크리스트의 할 일을 클릭하면 **완료 표시(체크박스) + 취소선**으로 바뀐다(텍스트 클릭도 동작)
- [ ] 다시 클릭하면 원래대로 되돌아간다(토글)
- [ ] **키보드만으로**: `Tab`으로 할 일 체크박스에 포커스가 이동하고, `Space`로 완료/해제가 토글된다 (v0.4.1)
- [ ] 요약 헤더 아래 **진행률 바**(완료/전체 개수)가 표시되고, 클릭할 때마다 실시간 반영된다
- [ ] 체크리스트가 갱신(수정 요청 반영)되면 진행률이 새 전체 개수 기준으로 재계산된다
- [ ] 우측 패널 상단 **"복사"** 버튼 → 클립보드 복사 시도, 성공 시 "복사됨"으로 잠시 바뀐다
  (HTTPS가 아닌 배포 환경에서도 실패로 조용히 멈추지 않고 "복사 실패" + **대안 안내 봇 메시지**
  ("다운로드 버튼 사용 / 직접 선택 복사")가 표시되는지 확인 · v0.4.1)
- [ ] **"다운로드"** 버튼 → `delaynomore-<목표명>.txt` 파일이 받아지고, 내용에 목표/기간/Day별 할 일이 포함된다

## F-4. 빠른 동작 (v0.4.0 기준으로 갱신)
- [ ] 계획 생성 후 대화 입력창 위에 **"계획 저장(고정)" · "기간 +3일" · "처음부터 다시 만들기"** 버튼이 보인다
- [ ] **"계획 저장(고정)"** 클릭 → 확인 창 후 계획이 **고정(CONFIRMED)** 된다 — 헤더에 "고정됨" 배지,
      대화로 수정 요청 시 반영되지 않고 잠금 안내, "기간 +3일" 버튼은 숨겨지고 완료 체크만 가능
- [ ] (고정 후) 대화에 **`주말은 빼줘`** 입력 → "분석 중" 표시·AI 호출 없이 🔒 안내가 **한 번만** 뜬다 (v0.4.1)
- [ ] (고정 후) 대화에 **질문**(예: "Day 2는 왜 이렇게 짰어?") → 차단되지 않고 정상 답변된다 (v0.4.1)
- [ ] **"기간 +3일"** 클릭(고정 전) → 챗 흐름을 통해 자동 요청이 전송되고, **기존 Day는 내용이 바뀌지 않은 채**
      새 Day 3개가 뒤에 이어붙는다(총 기간 숫자도 갱신)
- [ ] **"처음부터 다시 만들기"** 클릭 → 확인 창("현재 계획은 보관함에 남고...") 후 슬롯필링 첫 질문으로
      돌아가되, **이전 계획은 보관함 목록에 그대로 남는다**

## F-5. 여러 계획 보관 (v0.4.0) — 서버 공유 보관함
> 계획은 서버 인메모리에 보관된다: **모든 방문자가 같은 목록을 보고, 서버 재시작 시 초기화**된다(로그인/DB는 후속 단계).
- [ ] 계획 초안이 완성되면 자동으로 보관되어, 우측 패널 상단에 **"보관된 계획 N개"** 바가 나타난다
- [ ] 바 클릭 → 목록 펼침: 목표명 · 기간/시간 · 완료 진행률 · 저장일이 보이고, 현재 계획엔 **"보는 중"**(파란 테두리) 표시
- [ ] "처음부터 다시 만들기" → 새 계획 생성 → 목록이 **2개**가 된다(이전 계획 유지)
- [ ] 다른 계획 클릭 → 체크리스트/대화가 그 계획으로 전환된다(대화창엔 안내 말풍선 1개)
- [ ] 목록을 **펼친 상태**에서 현재 계획의 할 일을 체크 → 본문 진행 바와 목록의 "보는 중" 행 진행률이 **즉시 같은 값**이 된다 (v0.4.1)
- [ ] 할 일 완료 토글 후 잠시 뒤 **다른 브라우저/기기**에서 목록을 열면 진행률이 반영되어 보인다(공유 저장소)
- [ ] 계획을 고정하면 다른 브라우저의 목록에도 **잠금 아이콘**이 표시된다
- [ ] 휴지통 클릭 → 확인 창("모든 방문자의 목록에서 사라집니다") 후 삭제되고, 다른 브라우저에서도 목록을 다시 열면 사라져 있다
- [ ] 다른 방문자가 이미 지운 계획을 클릭 → "이미 삭제된 계획입니다" 안내 + 목록 갱신
- [ ] 계획 A를 보던 중 새로고침 → A가 자동 복원된다("서버 보관함에서 불러왔습니다")
- [ ] **서버 재시작** → 목록이 비어 있고(휘발성), 앱은 새 계획 생성부터 정상 동작한다
- [ ] **백엔드 중지** 상태로 접속 → 목록 바에 "불러오지 못했습니다 + 다시 시도"가 뜨지만, 계획 생성(mock)·대화는 계속 동작한다

## F-6. 오늘 할 일 / 오늘 보기 (v0.5.0)
> 화면 상단(헤더 아래) 접이식 밴드. **보관된 계획**에서 오늘 날짜의 할 일만 모아 보여준다. 보관 계획이 없으면 밴드 자체가 숨겨진다.
- [ ] 계획을 **2개 이상** 저장(보관)하면, 상단 "오늘 할 일" 밴드에 두 계획의 오늘 할 일이 **계획 이름과 함께 병합** 표시된다
- [ ] 밴드 머리에 **"오늘 n/m 완료"** 진행률이 표시되고, 체크할 때마다 즉시 갱신된다
- [ ] 밴드에서 할 일 체크 → **원래 계획(우측 체크리스트/진행 바)에 즉시 반영**된다. 우측 체크리스트에서 체크해도 밴드에 즉시 반영된다(양방향)
- [ ] **다른(보는 중이 아닌) 계획**의 할 일을 밴드에서 체크 → **새로고침 후에도** 완료 상태가 유지되고, **다른 브라우저/기기**에서도 같은 상태가 보인다(서버 저장)
- [ ] 고정(CONFIRMED)된 계획은 밴드에 **🔒 아이콘**이 보이고, 완료 체크는 계속 가능하다
- [ ] **"계획 보기"** 버튼 클릭 → 우측 패널이 해당 계획으로 전환된다(보는 중 계획이면 변화 없음 · "보는 중" 표시)
- [ ] 계획을 **삭제**하면 밴드에서 그 계획의 오늘 항목이 사라진다
- [ ] 모든 보관 계획에 오늘 항목이 없으면(기간이 지났거나 시작 전) **"오늘 할 일이 없습니다"** 안내가 보인다
- [ ] 밴드 접기/펼치기가 동작하고, **펼칠 때 보관함을 다시 불러와** 다른 방문자의 변경이 반영된다
- [ ] 모바일 폭(<760px)에서도 밴드가 분할 화면 **위에** 표시된다

## G. 반응형
- [ ] 브라우저 폭을 좁히거나 모바일 접속 → 좌우 → **위아래 스택**으로 전환

## H. 배포 / 운영 (VM에서)
- [ ] `sudo docker ps` → `delaynomore` **Up** (Restarting 반복 아님 = 메모리 OK)
- [ ] VM 재부팅 후에도 자동 기동 (`--restart unless-stopped`)
- [ ] `curl -s http://localhost/api/v1/ai/health` 정상 응답
- [ ] `curl -s http://localhost/api/v1/plans` → `{"success":true,"data":[...]}` 정상 응답(보관함 목록)

## 참고
- **키 없이 배포**한 상태면 E·F 결과는 **템플릿 mock**이라 내용이 정형화되어 있다. 실제 AI 문장을 보려면 키를 넣고 재배포
  (`OPENROUTER_API_KEY=... ./deploy/oci-setup.sh`) 후 헤더가 "AI 연결됨"인지 확인하고 E·F를 다시 테스트한다.
- 이상 동작 발견 시: 재현 입력값과 화면 상태를 기록해 이슈로 남기고 다음 패치에 반영한다.
