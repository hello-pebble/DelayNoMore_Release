# 기능 점검 체크리스트 (QA)

- **대상 버전**: `v0.8.0`
- **범위**: 대화형 투두리스트 생성 데모 — 가로 3칸 UI(대화/오늘 할 일/체크리스트), 슬롯필링, 계획 생성/수정, mock 폴백, 여러 계획 보관(서버 공유 보관함), 오늘 할 일(오늘 보기), 오늘 마무리(일일 회고), 미완료 항목 내일로 이동, 계획 변경 이력(Audit), 서버 규칙 강제(고정 가드·형식 검증)
- **사용법**: 배포 URL(`http://<PUBLIC_IP>` 또는 배포 주소)에 접속해 아래 항목을 순서대로 확인.
- **버전 갱신 시**: 새 버전에서 바뀐 항목을 추가하고 "대상 버전"을 올린다.

## A. 화면 / 레이아웃
- [ ] 가로 3칸 구성이 보인다 — 왼쪽 **대화**, 가운데 **오늘 할 일**, 오른쪽 **체크리스트** (v0.5.1)
- [ ] 헤더에 `DelayNoMore` + 우측 **AI 상태 표시**(점 + 텍스트)
- [ ] 초기 체크리스트 패널: "왼쪽 대화에서 … 입력하면 이곳에 계획표가 생성됩니다" 안내문
- [ ] 첫 진입 시 봇이 **첫 질문(목표)** 을 자동으로 발송

## B. AI 연결 상태 (헤더)
- [ ] 키 없이 배포 → **"API Key 미설정"**(주황) 표시 (생성은 mock 폴백)
- [ ] 키를 넣고 배포 → **"AI 연결됨"**(초록)

## C. 슬롯필링 대화 (순서대로 4개 질문)
- [ ] **목표** 입력 → 다음으로 **기간** 질문
- [ ] **기간** 질문 시 **3일/5일/7일 빠른 선택 버튼**이 **질문 말풍선 바로 아래(대화 영역 안)** 에 뜨고,
      클릭하면 바로 전송된다(직접 숫자 입력도 가능)
- [ ] **기간** 입력(버튼 또는 직접) → **하루 투자 시간** 질문 + **1/2/4/6시간 버튼**
- [ ] **하루 시간** 입력 → **현재 수준** 질문 + **완전 초보/기본 개념은 아는 수준/실전 경험 있음 버튼**
- [ ] 유저 메시지는 오른쪽 정렬(파란 말풍선), 봇은 왼쪽(회색)

## D. 입력 검증 (일부러 틀리게 입력)
- [ ] 목표에 `a` 한 글자 → "더 구체적으로" 경고
- [ ] 기간에 `0` 또는 `50` → "1~14일" 경고
- [ ] 기간/시간에 `abc`(숫자 아님) → 경고
- [ ] 하루 시간에 `30` → "1~24시간" 경고
- [ ] 수준에 한 글자 → 경고

## E. 체크리스트 생성 (핵심)
정상 입력 예: 목표 `React 기초 공부` / 기간 `3` / 시간 `2` / 수준 `완전 초보`
- [ ] 마지막 입력 후 **"분석 중… (n초)"** thinking 표시가 뜬다
- [ ] 체크리스트 패널에 **제목 + 요약(기간·시간·수준)** 헤더 생성
- [ ] **Day 1 / Day 2 / Day 3** 카드가 날짜와 함께 표시
- [ ] 각 Day의 할 일 개수가 **하루 투자 시간에 비례**(1h→2개, 2h→3개, 3~4h→4개, 5h+→5~6개 안팎)
- [ ] 날짜가 **오늘부터** 순서대로 맞다

## F. 수정 루프 (초안 생성 후)
- [ ] 입력창 placeholder가 "수정 요청이나 질문을…"으로 바뀐다
- [ ] `주말은 빼줘` 입력 → 재생성 → 주말 처리 반영
- [ ] `일정을 늘려줘` 입력 → 할 일 개수/강도 증가
- [ ] 재생성 중에도 thinking 표시가 뜨고, 끝나면 체크리스트 갱신

## F-2. 자유 대화 (v0.1.0 포함, AI 키 연결 상태에서 확인 권장)
- [ ] 수정 요청 시 답변이 고정 문구가 아니라 **무엇을 어떻게 바꿨는지** 구체적으로 설명한다
- [ ] `공부 항목 좀 더 구체적으로` → 할 일 문구가 실제로 구체화되고, 답변에 변경 내용이 언급된다
- [ ] 계획에 대한 **질문**(예: "Day 2는 왜 이렇게 짰어?") → 계획을 다시 만들지 않고 자연어로 답한다
- [ ] `?` 같은 **불명확한 입력** → "반영했습니다" 대신 예시와 함께 되묻는다
- [ ] 직전 요청이 반영 안 됐을 때 `반영 안됐는데?` → 대화 이력을 참고해 그 요청을 다시 반영한다
- [ ] (키 없이 배포) 인식 못 하는 요청 → "(오프라인 모드)" 안내와 함께 가능한 요청 예시를 보여준다

- [ ] 요약 헤더 아래 **진행률 바**(완료/전체 개수)가 표시되고, 클릭할 때마다 실시간 반영된다
- [ ] 체크리스트가 갱신(수정 요청 반영)되면 진행률이 새 전체 개수 기준으로 재계산된다

## F-4. 빠른 동작 (v0.4.0 기준으로 갱신)
- [ ] 계획 생성 후 대화 입력창 위에 **"계획 저장(고정)" · "기간 +3일" · "처음부터 다시 만들기"** 버튼이 보인다
- [ ] **"계획 저장(고정)"** 클릭 → 확인 창 후 계획이 **고정(CONFIRMED)** 된다 — 헤더에 "고정됨" 배지,
      대화로 수정 요청 시 반영되지 않고 잠금 안내, "기간 +3일" 버튼은 숨겨지고 완료 체크만 가능
- [ ] (고정 후) 대화에 **`주말은 빼줘`** 입력 → "분석 중" 표시·AI 호출 없이 🔒 안내가 **한 번만** 뜬다 (v0.4.1)
- [ ] (고정 후) 대화에 **질문**(예: "Day 2는 왜 이렇게 짰어?") → 차단되지 않고 정상 답변된다 (v0.4.1)
- [ ] **"기간 +3일"** 클릭(고정 전) → 챗 흐름을 통해 자동 요청이 전송되고, **기존 Day는 내용이 바뀌지 않은 채**
      새 Day 3개가 뒤에 이어붙는다(총 기간 숫자도 갱신)
- [ ] **"처음부터 다시 만들기"** 클릭 → 확인 창("현재 계획은 보관함에 남고...") 후 슬롯필링 첫 질문으로
      돌아가되, **이전 계획은 보관함 목록에 그대로 남는다**

## F-5. 여러 계획 보관 (v0.4.0) — 서버 공유 보관함
> 계획은 서버 인메모리에 보관된다: **모든 방문자가 같은 목록을 보고, 서버 재시작 시 초기화**된다(로그인/DB는 후속 단계).
- [ ] 계획 초안이 완성되면 자동으로 보관되어, 체크리스트 패널 상단에 **"보관된 계획 N개"** 바가 나타난다
- [ ] 바 클릭 → 목록 펼침: 목표명 · 기간/시간 · 완료 진행률 · 저장일이 보이고, 현재 계획엔 **"보는 중"**(파란 테두리) 표시
- [ ] "처음부터 다시 만들기" → 새 계획 생성 → 목록이 **2개**가 된다(이전 계획 유지)
- [ ] 다른 계획 클릭 → 체크리스트/대화가 그 계획으로 전환된다(대화창엔 안내 말풍선 1개)
- [ ] 목록을 **펼친 상태**에서 현재 계획의 할 일을 체크 → 본문 진행 바와 목록의 "보는 중" 행 진행률이 **즉시 같은 값**이 된다 (v0.4.1)
- [ ] 할 일 완료 토글 후 잠시 뒤 **다른 브라우저/기기**에서 목록을 열면 진행률이 반영되어 보인다(공유 저장소)
- [ ] 계획을 고정하면 다른 브라우저의 목록에도 **잠금 아이콘**이 표시된다
- [ ] 휴지통 클릭 → 확인 창("모든 방문자의 목록에서 사라집니다") 후 삭제되고, 다른 브라우저에서도 목록을 다시 열면 사라져 있다
- [ ] 다른 방문자가 이미 지운 계획을 클릭 → "이미 삭제된 계획입니다" 안내 + 목록 갱신
- [ ] 계획 A를 보던 중 새로고침 → A가 자동 복원된다("서버 보관함에서 불러왔습니다")
- [ ] **서버 재시작** → 목록이 비어 있고(휘발성), 앱은 새 계획 생성부터 정상 동작한다
- [ ] **백엔드 중지** 상태로 접속 → 목록 바에 "불러오지 못했습니다 + 다시 시도"가 뜨지만, 계획 생성(mock)·대화는 계속 동작한다

## F-6. 오늘 할 일 / 오늘 보기 (v0.5.0 · 배치 v0.5.1)
> 화면 가운데 상시 칸. **보관된 계획**에서 오늘 날짜의 할 일만 모아 보여준다. 보관 계획이 없으면 "계획을 만들면 오늘 할 일이 여기에 모여요" 안내가 보인다.
- [ ] 계획을 **2개 이상** 저장(보관)하면, 가운데 "오늘 할 일" 칸에 두 계획의 오늘 할 일이 **계획 이름과 함께 병합** 표시된다
- [ ] 칸 머리에 **"오늘 n/m 완료"** 진행률이 표시되고, 체크할 때마다 즉시 갱신된다
- [ ] 오늘 칸에서 할 일 체크 → **원래 계획(오른쪽 체크리스트/진행 바)에 즉시 반영**된다. 오른쪽 체크리스트에서 체크해도 오늘 칸에 즉시 반영된다(양방향)
- [ ] **다른(보는 중이 아닌) 계획**의 할 일을 오늘 칸에서 체크 → **새로고침 후에도** 완료 상태가 유지되고, **다른 브라우저/기기**에서도 같은 상태가 보인다(서버 저장)
- [ ] 고정(CONFIRMED)된 계획은 오늘 칸에 **🔒 아이콘**이 보이고, 완료 체크는 계속 가능하다
- [ ] **"계획 보기"** 버튼 클릭 → 체크리스트가 해당 계획으로 전환된다(보는 중 계획이면 변화 없음 · "보는 중" 표시)
- [ ] 계획을 **삭제**하면 오늘 칸에서 그 계획의 오늘 항목이 사라진다
- [ ] 모든 보관 계획에 오늘 항목이 없으면(기간이 지났거나 시작 전) **"오늘 할 일이 없습니다"** 안내가 보인다
- [ ] 칸 머리의 **새로고침 버튼**을 누르면 보관함을 다시 불러와 다른 방문자의 변경이 반영된다 (v0.5.1)
- [ ] 모바일 폭(<760px)에서는 대화/오늘 할 일/체크리스트 순서의 위아래 스택으로 전환되고, 오늘 칸이 화면을 다 차지하지 않는다(최대 40vh) (v0.5.1)

## F-7. 오늘 마무리 / 일일 회고 (v0.6.0)
> 오늘 할 일 칸 **하단의 접이식 "오늘 마무리" 섹션**. 회고는 계획별·오늘 날짜 1건(재저장 시 갱신)이고, 계획과 같은 휘발성 공유 저장소를 쓴다. 오늘 항목이 있는 계획이 없으면 섹션 자체가 보이지 않는다.
- [ ] 오늘 할 일 5개 중 3개를 체크 → **"오늘 마무리"** 바 클릭 → 계획 카드에 **"오늘 5개 중 3개 완료 · 완료율 60%"** 가 자동 표시된다(직접 입력 없음)
- [ ] **벅찼어요** + **시간이 부족했어요** 선택 → "회고 저장" 클릭 → **"오늘 회고를 저장했습니다."** + `3/5 완료 · 벅찼어요 · 시간이 부족했어요` 요약이 보인다
- [ ] 난이도·이유 중 **하나만 선택**한 상태에서는 "회고 저장" 버튼이 비활성이다
- [ ] **새로고침** 후 "오늘 마무리"를 다시 펼치면 저장한 회고가 그대로 복원된다(서버 저장)
- [ ] **다른 브라우저/기기**(또는 오늘 칸 새로고침 버튼)에서도 같은 회고가 보인다(공유 저장소)
- [ ] 저장 후 할 일을 **1개 더 체크** → 카드의 라이브 카운트가 4/5로 갱신되고, **"완료 상태가 바뀌었어요 · 다시 저장하면 4/5로 갱신됩니다"** 안내가 뜬다
- [ ] **"수정"** 클릭 → 저장했던 선택이 프리필된 폼이 열리고, 다시 저장하면 **중복 생성 없이** 같은 날짜 회고가 갱신된다(완료 개수도 서버 재계산 값으로)
- [ ] **고정(CONFIRMED) 계획**에서도 회고 저장이 동작한다(완료 체크와 같은 취급)
- [ ] 회고를 저장한 계획을 **삭제** → 회고도 함께 삭제된다(같은 planId로 회고 조회 시 404)
- [ ] API 직접 호출: 오늘이 아닌 날짜로 PUT → 400 `REFLECTION_DATE_NOT_TODAY`, `abc` 날짜 → 400 `REFLECTION_DATE_INVALID`, 잘못된 enum → 400 `INVALID_INPUT`, 없는 계획 → 404 `PLAN_NOT_FOUND`, 저장 전 GET → 404 `REFLECTION_NOT_FOUND`
- [ ] **백엔드 재시작** → 회고가 사라지고(휘발성 정상), 보관함이 비면 섹션도 자연히 사라지며 기존 휘발성 안내가 커버한다

## F-8. 미완료 항목 내일로 이동 (v0.7.0)
> 오늘 할 일 칸의 계획 그룹마다, **DRAFT(고정 전) 계획**에 오늘 미완료가 있으면 "미완료 N건 내일로" 버튼이 보인다. 고정(CONFIRMED) 계획은 계획 구조를 바꾸는 행동이라 버튼 자체가 숨겨진다.
- [ ] 오늘 할 일이 있는 DRAFT 계획 → 그룹 하단에 **"미완료 N건 내일로"** 버튼이 보이고, N이 실제 미완료 개수와 같다
- [ ] 버튼 클릭 → **확인 창**("미완료 N건을 내일(YYYY-MM-DD)로 옮길까요?") 후 이동된다 — 취소하면 아무 변화 없음
- [ ] 이동 후: 오늘 그룹에서 **미완료만 사라지고 완료 항목은 남는다**(전부 미완료였다면 그룹 자체가 사라짐)
- [ ] 오른쪽 체크리스트에서 **내일 Day 카드 뒤에** 이동된 항목이 붙어 있고, 오늘 Day에는 완료 항목만 남는다(오늘이 비면 Day 카드도 사라짐)
- [ ] **마지막 날**의 미완료를 이월 → 내일 Day가 새로 생기고 요약 헤더의 **"기간 N일"이 +1** 된다
- [ ] **고정(CONFIRMED) 계획**은 오늘 미완료가 있어도 버튼이 **안 보인다**(완료 체크는 계속 가능)
- [ ] 미완료가 **0건**이면(모두 완료) 버튼이 안 보인다
- [ ] **보는 중이 아닌** 계획도 이월 동작하고, 새로고침·**다른 브라우저/기기**에서도 이동 결과가 유지된다(서버 저장)
- [ ] 이월 직후 그 계획의 **변경 이력**에 `계획 수정 · 미완료 N건을 <내일 날짜>로 이동`이 기록된다
- [ ] 다른 방문자가 이미 지운 계획에서 이월 시도 → "이미 삭제된 계획입니다" 안내 + 목록 갱신

## F-9. 계획 변경 이력 / Audit (v0.7.0)
> 보관된 계획 행의 **이력 버튼(🕘) 하나**로 그 계획의 변경 기록을 본다. 이벤트는 서버가 변경 시점에 diff로 판별해 직접 기록하며(7종), 읽기 행동·AI 대화 전문은 기록하지 않는다. 계획과 같은 휘발성 저장소(전역 최근 1,000건 상한).
- [ ] "보관된 계획" 목록의 각 행에 **이력 버튼(🕘)** 이 보인다 — 클릭 시 행 아래로 패널이 펼쳐지고, 다시 클릭하면 접힌다
- [ ] 계획 생성 직후 이력: **"계획 생성 · 이 브라우저 · 방금 전"** 1건
- [ ] 할 일 체크 → (600ms 동기화 후 재펼침) **"할 일 완료 · "내용" · 날짜"**, 해제 → **"완료 해제"** — 빠르게 여러 개 체크해도 **건별로** 기록된다
- [ ] "계획 저장(고정)" → 이력에 **"계획 고정"**
- [ ] 대화로 계획 수정(예: "주말은 빼줘") → **"계획 수정 · 계획 내용 변경"**
- [ ] 회고 저장 → **"회고 저장 · <날짜> 회고 저장 (M/N 완료)"**
- [ ] **다른 브라우저/기기**에서 같은 계획을 체크 → 이력을 다시 펼치면 그 변경에 **"다른 세션"** 배지가 붙는다(내 변경은 "이 브라우저")
- [ ] `X-Session-Id` 헤더 없이 API로 직접 PUT(curl) → 해당 이벤트의 배지가 **"알 수 없음"**
- [ ] 이력 패널을 **펼쳤다 접기만** 반복해도 새 이벤트가 생기지 않는다(읽기는 기록 안 함)
- [ ] 계획 **삭제** 후 `curl http://localhost/api/v1/plans/<id>/audit-events` → **404가 아니라** `PLAN_DELETED` 포함 전체 이력이 남아 있다(이력은 캐스케이드 삭제 안 함)
- [ ] 없는 planId로 조회 → 오류가 아니라 **빈 배열**
- [ ] **백엔드 중지** 상태에서 이력 펼침 → "이력을 불러오지 못했습니다 + 다시 시도" 표시, 재기동 후 다시 시도가 동작한다
- [ ] **서버 재시작** → 이력이 초기화되어 "기록이 없습니다 (서버 재시작 시 이력도 초기화됩니다)" 안내가 보인다(휘발성 정상)
- [ ] 패널 하단에 "모든 방문자가 함께 보는 데모 이력입니다 · 최근 1000건만 보관됩니다" 안내가 보인다

## F-10. 고정 계획 서버 가드 / PLAN_LOCKED (v0.8.0)
> 예전엔 프론트만 잠금을 지켰고 서버는 그대로 저장했다(last-write-wins). 이제 서버가 강제한다: 고정(CONFIRMED) 계획은 **완료 체크만 허용**, 구조 변경은 409 `PLAN_LOCKED`. 브라우저 UI로는 애초에 수정이 차단되므로, 가드 자체는 **curl로 직접 확인**한다.

**브라우저 (기존 흐름 회귀 — 가드가 정상 경로를 막지 않는지)**
- [ ] DRAFT 계획: 대화 수정·기간 +3일·이월이 기존대로 동작한다(가드 미적용)
- [ ] 수정 직후 바로 "계획 저장(고정)" 클릭(600ms 디바운스 안에 수정+고정이 한 PUT으로 배칭) → 정상 고정된다
- [ ] 고정 후: 오른쪽 체크리스트·오늘 칸에서 **완료 체크/해제가 정상 동작**하고, 새로고침·다른 브라우저에서도 유지된다
- [ ] 고정 후 완료 토글 → 변경 이력에 **"할 일 완료"** 가 기존대로 기록된다
- [ ] 고정된 계획 **삭제**는 여전히 가능하다(탈출구 유지)
- [ ] (두 브라우저 경합) A에서 DRAFT 계획을 보며 수정 입력 중, B에서 같은 계획을 먼저 고정 → A의 수정은 서버에 반영되지 않고(콘솔 경고 후 대기 변경 폐기), A가 계획을 다시 불러오면 고정 상태로 보인다
- [ ] (두 브라우저 경합) B가 먼저 고정한 계획을 A의 오늘 칸에서 이월 시도 → **"다른 세션에서 고정된 계획이라 이월할 수 없습니다"** 안내 + 목록 갱신

**curl (가드 직접 확인 — `<id>`는 고정된 계획)**
- [ ] 고정 계획에 goalName 변경 PUT → **409** + `error.code = PLAN_LOCKED`
- [ ] 고정 계획에 `status: "DRAFT"` 롤백 PUT → **409**
- [ ] 고정 계획에 duration/endDate +1 (이월 흉내) PUT → **409**
- [ ] 고정 계획에 할 일 content 변경/항목 추가 PUT → **409**
- [ ] 고정 계획에 confirmedAt만 바꾼 PUT → **409**
- [ ] 고정 계획에 **completed만 플립**한 PUT → **200**, 이력에 `TASK_COMPLETED`
- [ ] 동일 본문 재PUT(no-op) → **200**, 새 이력 없음
- [ ] 409로 거부된 뒤 GET → 계획이 **원상태 그대로**이고, 거부된 변경은 **이력에 없다**
